**Raspberry Pi & Radar Subsystems**
This folder contains all of the code that is needed to be run on the raspberry pi. For this project, a raspberry pi 5 SBC was used along with a IWR1642BOOST evaluation radar board from TI. The two were connected usign a micro-USB to USB connector. The raspberry pi in this project acts as the mediators between all other subsystems and the central point where everything is run (using threaded runners).

If the goal is to run the code while the pi is connected to either a laptop via ssh or plugged into a monitor and have the output on a console, the main requirements are that the system is correctly wired up and all of the files (in this folder) are in one directory in the rapsberry pi. Additionally, it is necessary that the folder contains both a .onnx ML file with the model you are using, along with the associated scaler.save and requirements.txt files. From there naviagte to main.py and run the program. If it is done correctly, it will show the radar board being programmed, and after 20s fall/no fall outputs should begin in the terminal in 5s incrments.

To run the program off boot and restart (as would be intended for the actual product), edit the myapp.service file to correctly reflect the location of main and the correct raspberry pi user. Then open up systemd using sudo cp myapp.service /etc/systemd/system in the command line before restarting: sudo systemctl restart myapp.service. From there the program should work as intended and run off of boot on the raspberry pi. To check the status of it in the command line, use systemctl status myapp.service.
